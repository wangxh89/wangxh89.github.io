RequireJS采用模块化定义,避免全局变量,实现AMD(Asynchronous Module Definition)异步模块加载,RequireJS解决下面两个问题

  • 首先，加载的时候，浏览器会停止网页渲染，加载文件越多，网页失去响应的时间就会越长。RequireJS实现js文件的异步加载，避免网页失去响应；

  • 由于js文件之间存在依赖关系，因此必须严格保证加载顺序（比如上例的1.js要在2.js的前面），依赖性最大的模块一定要放到最后加载，当依赖关系很复杂的时候，代码的编写和维护都会变得困难。 RequireJS管理模块之间的依赖性，便于代码的编写和维护

require使用

  • data-main 属性，定义RequireJS启动脚本，RequirejsBase.js是异步加载

     <script type="text/javascript" data-main="RequirejsBase.js"
         src="../javascripts/requirejs/require.js"></script>

  • RequirejsBase.js

     require.config({
     baseUrl : '../javascripts/',
     config : {
         log : {level : 'debug'}
     },
     paths : {
         log : 'requirejs/plugins/log',
         domReady : 'requirejs/plugins/domReady',
         jquery : 'jquery/jquery-1.8.3'
     }
 });

 require(['log!requirejsbase','domReady','jquery'], function(log,domReady,$) {
     domReady(function() {
         log.debug('Main dom Ready!');
         log.debug("This this debug msg");
         log.info("This this info msg");
         log.warn("This this warn msg");
         log.error("This this error msg");
     })

 });
    1. baseUrl查找模块文件根目录，如果没有配置则使用加载requirejs的HTML页面所在目录
    2. paths 用于映射不能在baseUrl下找到的模块名称，设置路径是相对于baseUrl的。如果以”/“或者ul(如”http:”)开头，则不是

  • 加载非规范的模块

    理论上，require.js加载的模块，必须是按照AMD规范、用define()函数定义的模块。但是实际上，虽然已经有一部分流行的函数库（比如jQuery）符合AMD规范，更多的库并不符合。underscore个库，都没有采用AMD规范编写。如果要加载它们的话，必须先定义它们的特征。

     require.config({
     baseUrl : '../javascripts/',
     config : {
         log : {level : 'debug'}
     },
     paths : {
         log : 'requirejs/plugins/log',
         domReady : 'requirejs/plugins/domReady',
         jquery : 'jquery/jquery-1.8.3',
         fishError : 'requirejs/plugins/FishError',
         underscore : 'underscore'
     },
     'shim' : {
         'underscore' : {
             'exports' : '_'
         }
     }
 });

    require.config()接受一个配置对象，这个对象除了有前面说过的paths属性之外，还有一个shim属性，专门用来配置不兼容的模块。具体来说，每个模块要定义（1）exports值（输出的变量名），表明这个模块外部调用时的名称；（2）deps数组，表明该模块的依赖性。
    比如，jQuery的插件可以这样定义：

     shim: {
 　　　　'jquery.scroll': {
 　　　　　　deps: ['jquery'],
 　　　　　　exports: 'jQuery.fn.scroll'
 　　　　}
 　　}

模块的写法

require.js加载的模块，采用AMD规范。也就是说，模块必须按照AMD的规定来写。
具体来说，就是模块必须采用特定的define()函数来定义。如果一个模块不依赖其他模块，那么可以直接定义在define()函数之中。

define(['log!index','minError'],function(log,fishError){
    var IndexMgr = {
        init : function() {
            log.debug('Index init success');
            $('#btnlog').click(IndexMgr.btnLogClick);
            $('#btnerror').click(IndexMgr.btnErrorClick);
        },
        btnLogClick : function() {
            log.error("Log btn click");
        },
        btnErrorClick: function () {
            var error = new minError('index');
            throw error('minError-000','params a is {0}',1);
        }
    };
    return IndexMgr;
})

require.js插件

require.js还提供一系列插件，实现一些特定的功能。

  • domready插件，可以让回调函数在页面DOM结构加载完成后再运行。

  • text! 文本插件，自动处理以text!为前缀模块。github:https://github.com/requirejs/text

  • i18n! 国际化插件。根据配置的语言，如果没有配置根据navigator.language来寻找资源文件

  • css! github:https://github.com/guybedford/require-css

  • tpl! underscore模板插件。github:https://github.com/dciccale/requirejs-underscore-tpl
