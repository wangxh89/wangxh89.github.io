关于IOS追踪用户设备总结

一、 完全不能使用方案　

1、UDID

设备唯一标识符（Unique Device Identifier）之前被各种国内外统计平台，应用开发商广泛使用，后Apple从2013年05月01日起拒绝接受使用UDID的应用后，立毙！并且iOS7中已经完全的禁用了它

https://developer.apple.com/library/ios/releasenotes/iOS70APIDiffs/index.html#//apple_ref/doc/uid/TP40013203
https://developer.apple.com/library/ios/releasenotes/General/iOS70APIDiffs/index.html#//apple_ref/doc/uid/TP40013203
Removed UIDevice.uniqueIdentifier

2、 Mac地址

苹果并不希望有人通过MAC地址来分辨用户，所以如果你在iOS7系统上查询MAC地址，它现在只会返回02:00:00:00:00:00。

https://developer.apple.com/library/iOS/releasenotes/General/RN-iOSSDK-7.0/index.html

Networking
Notes

Two low-level networking APIs that used to return a MAC address now return the fixed value 02:00:00:00:00:00. The APIs in question are sysctl (NET_RT_IFLIST) and ioctl (SIOCGIFCONF). Developers using the value of the MAC address should migrate to identifiers such as -[UIDevice identifierForVendor]. This change affects all apps running on iOS 7.

3、 第三方解决方案OPENUDID

IOS7 UIPasteboard由共享变为沙盒化了 这样就让OpenUDID不那么有用了。

http://blog.appsfire.com/udid-is-dead-openudid-is-deprecated-long-live-advertisingidentifier/

二、 各有缺点的方案

1、 CFUUID

就是将用户自己生成唯一标识字符串，然后将唯一标识字符串存放到NSUserDefaults或者keychain中

CFUUID 问题：

A、每次生成的唯一标识字符串是不一样的，存在NSUserDefaults应用删除后就读取不到了。 存在keychain中必须是用同样证书的应用才能读取。

B、如果你从一台旧设备中备份文件到新设备中，两台设备就拥有相同的 CFUUID。如果你从临时文件中备份操作系统，就会出现一个设备里存在不同 CFUUID 的情况。

2、 DeviceToken

这是推送用的令牌，是唯一的，但用户如果没开推送，或者拒绝了，这个就没有了

3、 Apple建议方案

新增了两个用于替换uniqueIdentifier的接口，分别是：

identifierForVendor： 当我们删除了某一个设备上某个开发商的所有APP之后，下次获取将会获取到不同的ID

advertisingIdentifier： 在同一个设备上的所有App都会取到相同的值，是苹果专门给各广告提供商用来追踪用户而设的，用户可以在 设置|隐私|广告追踪 里重置此id的值，或限制此id的使用，故此id有可能会取不到值。并且如果申请上架的应用里没有广告而使用这个ID的话，审核会不通过

三、 参考

苹果应用审查中关于隐私之一节

https://developer.apple.com/app-store/review/guidelines/#privacy
